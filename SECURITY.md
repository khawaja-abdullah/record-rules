# Security Policy

## Supported Versions

We take the security of RecordRules seriously. Currently, only the latest major release receives security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | ✅ Yes             |
| < 1.0.0 | ❌ No              |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a potential security vulnerability in this project, please notify us privately so we can address it before it is exploited.

### How to Report
You can report security concerns through either of the following channels:

1.  **Email:** Send a detailed report to **josephdanthikolla@gamil.com**.
2.  **LinkedIn:** Reach out directly to [Joseph Meghanath](https://www.linkedin.com/in/joseph-meghanath-9880ba149/).

### What to Include
To help us triage and fix the issue quickly, please include:
*   A description of the vulnerability.
*   Steps to reproduce the issue (PoC code or a sample Record definition).
*   The potential impact if exploited.

### Our Response Process
*   **Acknowledgment:** We will acknowledge receipt of your report within 48 hours.
*   **Investigation:** We will investigate the issue and may contact you for further details.
*   **Fix & Disclosure:** Once a fix is ready, we will release a new version. We follow a "coordinated disclosure" policy and ask that you do not share information about the vulnerability publicly until a patch has been merged and released.

## Safety First
We appreciate the efforts of security researchers who help keep open-source software safe. Thank you for your help!
